==================================================
2. ATTACHMENT EXTRACTION
==================================================

For each email source that supports attachments, extract:
- filename
- content type / mime type
- extension
- size if available
- content-id if embedded inline
- raw attachment bytes or saved file path
- sha256 hash
- embedded vs regular attachment

Support at minimum:
- .eml
- .msg
- Gmail export JSON structures if attachment payload is available
- any existing internal email structures already used in the project

If attachment binary content exists:
- save attachment file to a deterministic path
- use a safe filename
- avoid name collisions by prefixing with parent document id or attachment id

Example storage path:
attachments/<parent_document_id>/<safe_filename>

==================================================
3. ATTACHMENT TEXT PARSING
==================================================

Parse text from supported attachment types.

Support at minimum:
- pdf
- docx
- txt
- html
- csv
- json
- eml
- msg (if practical with existing parser flow)

Recommended libraries:
- pypdf for pdf
- python-docx for docx
- beautifulsoup4 for html
- standard json/csv/text parsers
- existing eml/msg parser stack for attached emails

For each supported attachment:
- extract text_content
- set text_extracted = true if successful
- generate a short text_summary
- record parse_method
- record warnings/errors

If unsupported:
- keep metadata only
- set text_extracted = false
- parse_status = "unsupported"

If parsing fails:
- keep metadata only
- set parse_status = "failed"
- record errors

Do not fail the email if attachment parsing fails.

This is strongly recommended for search and dashboard use.
