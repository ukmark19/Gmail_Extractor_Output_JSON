# Gmail Query Exporter

A production-quality web app for searching Gmail with advanced queries and exporting emails as structured JSON for ChatGPT knowledge base ingestion.

## Architecture

- **Frontend**: React + TypeScript + Tailwind CSS (Vite)
- **Backend**: Node.js + Express 5 + TypeScript
- **Database**: PostgreSQL (Drizzle ORM) — stores users, saved searches, export audit logs
- **Auth**: Google OAuth 2.0 (Gmail read-only scope)
- **Export**: Raw JSON, AI-optimized JSON, JSONL formats

## Setup Guide

### 1. Set up Google Cloud Project

1. Go to [Google Cloud Console](https://console.cloud.google.com/)
2. Create a new project (or use an existing one)
3. Navigate to **APIs & Services > Library**
4. Search for **Gmail API** and click **Enable**
5. Navigate to **APIs & Services > Credentials**
6. Click **Create Credentials > OAuth 2.0 Client IDs**
7. Set application type to **Web application**
8. Add authorized redirect URIs:
   - For local dev: `http://localhost/api/auth/google/callback`
   - For production: `https://your-app.replit.app/api/auth/google/callback`
9. Copy your **Client ID** and **Client Secret**

### 2. Configure OAuth Consent Screen

1. In Google Cloud Console, go to **APIs & Services > OAuth consent screen**
2. Set User Type to **External** (or Internal if using Google Workspace)
3. Fill in the app name, user support email, developer contact
4. Add scopes: `userinfo.email`, `userinfo.profile`, `gmail.readonly`
5. Add test users (your Gmail account) if in testing mode

### 3. Set Environment Variables

In Replit, go to **Secrets** and add:

| Key | Value |
|-----|-------|
| `GOOGLE_CLIENT_ID` | Your Google Client ID |
| `GOOGLE_CLIENT_SECRET` | Your Google Client Secret |
| `GOOGLE_REDIRECT_URI` | `https://your-app.replit.app/api/auth/google/callback` |

The `SESSION_SECRET` and database variables are already configured.

### 4. Run Locally

```bash
# Install dependencies
pnpm install

# Start the API server
pnpm --filter @workspace/api-server run dev

# Start the frontend (in a new terminal)
pnpm --filter @workspace/gmail-exporter run dev
```

## Export Formats

### A. Raw Structured JSON (`format: "raw"`)

Full Gmail data preserved:
```json
{
  "id": "msg123",
  "threadId": "thread456",
  "subject": "Invoice Q4",
  "from": "sender@example.com",
  "to": "you@example.com",
  "date": "Mon, 1 Jan 2024 10:00:00 +0000",
  "bodyPlainText": "...",
  "bodyHtmlStripped": "...",
  "labels": ["INBOX"],
  "attachmentMetadata": [{"filename": "invoice.pdf", "mimeType": "application/pdf", "size": 45000}],
  "queryUsed": "from:sender@example.com after:2024/01/01",
  "exportTimestamp": "2024-04-15T10:00:00.000Z"
}
```

### B. AI-Optimized JSON (`format: "ai-optimized"`)

Cleaned and normalized for LLM ingestion:
```json
{
  "document_id": "gmail_msg123",
  "source": "gmail",
  "subject": "Invoice Q4",
  "from": "sender@example.com",
  "content": "Subject: Invoice Q4\nFrom: sender@example.com\n\n[email body]",
  "summary_stub": "First 500 chars of content...",
  "metadata": { "gmail_id": "msg123", "has_attachments": true }
}
```

### C. JSONL (`format: "jsonl"`)

One AI-optimized JSON object per line — ideal for embedding workflows:
```
{"document_id":"gmail_msg1","content":"..."}
{"document_id":"gmail_msg2","content":"..."}
```

## Sample Gmail Queries Generated by UI

| UI Fields | Generated Query |
|-----------|----------------|
| From: boss@company.com, Subject: report | `from:boss@company.com subject:report` |
| Keywords: invoice, Date From: 2024-01-01, Has Attachment | `invoice after:2024/01/01 has:attachment` |
| From: newsletter@, Only Unread, Label: INBOX | `from:newsletter@ is:unread label:INBOX` |
| Exact Phrase: "quarterly review" | `"quarterly review"` |
| Does Not Have: unsubscribe, Date To: 2024-12-31 | `-unsubscribe before:2024/12/31` |
